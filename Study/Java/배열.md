## 배열(Array)

자바에서 배열은 다음과 같은 방식으로 선언할 수 있습니다.
```java
int[] array;
int array[];
```

두 방식 모두 문법적으로는 동일하지만, **타입에 배열임이 드러나는 첫 번째 방식(`int[] array`)이 일반적으로 더 선호됩니다**.  
배열 변수 선언 시에는 값이나 크기를 지정하지 않고, 대괄호만 사용하여 선언합니다.

선언 시 무조건 크기가 지정되어야만 하며, 한번 선언된 이후 배열의 크기를 증감시킬 수 없습니다.

배열을 실제로 사용하기 위해서는 반드시 초기화가 필요하며, 초기화는 다음과 같이 수행합니다.

```java
int[] array = new int[7];
```

이때 `new` 키워드를 사용하여 배열을 생성하고, 타입 이름과 함께 대괄호 안에 **배열의 크기**를 명시합니다.  
배열은 **참조 자료형**이므로, 새로운 배열을 생성할 때 반드시 `new` 키워드를 사용해야 합니다.

---

## 배열과 예외

배열의 인덱스 범위를 벗어난 위치에 값을 할당하거나, 해당 위치의 값을 참조하려고 하면  
`ArrayIndexOutOfBoundsException`이 발생합니다.  
이는 배열의 유효 범위는 `0`부터 `배열 길이 - 1`까지이기 때문입니다.

---

## 배열과 자료형

자바에서는 **모든 기본 자료형과 참조 자료형을 배열로 만들 수 있습니다**.  
기본 자료형으로 생성된 배열은 자동으로 기본값으로 초기화됩니다.

기본 자료형 배열의 초기값은 다음과 같습니다.

- `byte`, `short`, `int`, `long` 배열은 `0`으로 초기화됩니다.
- `float` 배열은 `0.0f`로 초기화됩니다.
- `double` 배열은 `0.0`으로 초기화됩니다.
- `char` 배열은 `'\u0000'`으로 초기화됩니다.
- `boolean` 배열은 `false`로 초기화됩니다.

참조 자료형으로 생성된 배열의 각 요소는 초기화하지 않으면 **`null` 값**을 가집니다.

## 배열 출력 시 형태

배열을 그대로 출력하면 실제 값이 아니라, 다음과 유사한 형태의 문자열이 출력됩니다.

```bash
[Ljava.lang.String;@12ef1wdq1
```

이 문자열을 통해 배열의 정보를 일부 유추할 수 있습니다.

문자열의 가장 앞에 있는 `[`는 해당 객체가 **배열임을 의미**합니다.  
그 다음의 `L`은 **참조 자료형 배열임을 의미**합니다.  
`java.lang.String`은 배열의 요소 타입을 나타냅니다.  
`@` 뒤에 나오는 값은 객체의 **고유 식별 값(hash code)**으로,  
참조 자료형에서 `toString()` 메소드를 재정의하지 않으면 이와 같은 형태로 출력됩니다.

기본 자료형 배열의 경우에는 다음과 같이 출력됩니다.

- `byte` 배열: `[B@고유번호`
- `short` 배열: `[S@고유번호`
- `int` 배열: `[I@고유번호`
- `long` 배열: `[J@고유번호`
- `float` 배열 : `[F@고유번호`
- `double` 배열 : `[D@고유번호`
- `char` 배열 : `[C고유번호`
- `boolean` 배열: `[Z@고유번호`

여기서 `long`과 `boolean`이 각각 `J`, `Z`로 표현되는 이유는,  
이미 참조 자료형인 `Long`, `Boolean`, `Byte` 등이 해당 문자를 사용하고 있기 때문입니다.

---

## 배열의 길이

배열의 길이는 메소드가 아닌 **필드**로 제공되며, `.length`를 통해 확인할 수 있습니다.

```java
array.length
```

---

## 배열 전용 반복문 (향상된 for문)

자바에는 배열을 순회하기 위한 전용 반복문이 존재합니다.  
이를 **향상된 for문(enhanced for loop)** 또는 **for-each 문**이라고 합니다.

구조는 다음과 같습니다.

```java
for (타입이름 임시변수명 : 반복대상객체) {
    // 실행 코드
}
```

이 반복문은 배열이나 컬렉션의 요소를 순차적으로 하나씩 꺼내어 처리할 때 사용하며,  
인덱스를 직접 다룰 필요가 없다는 장점이 있습니다.
