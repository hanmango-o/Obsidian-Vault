* 1. Kotlin의 주요 특성
* Koltin은 간결하고 실용적이며, Java코드와 상호 운용성(interoperability)을 중시
* Java가 사용중인 곳이라면 Koltin을 사용할 수 있음
* 1-1. 정적 타입 지원 언어
* Java와 마찬가지로 정적 타입(statically typed) 지원 언어임
* 정적 타입 지정 언어란 모든 프로그램 구성 요소의 타입을 컴파일 시점에 알 수 있고, 프로그램 안에서 객체의 필드나 메서드를 사용하 ㄹ떄마다 컴파일러가 타입을 검증해준다는 의미
* 반대로 동적 타입(dynamically typed)지정 언어는 타입과 관계 없이 모든 값을 변수에 넣을 수 있고, 메서드나 필드 접근에 대한 검증이 실행 시점에 일어남
	* 코드는 더 짧아지고 데이터 구조를 더 유연하게 가져갈 수 있다는 장점이 있지만 개발자의 실수도 컴파일 시 걸러내지 못해 실행 시점에 오류가 발생할 가능성이 있음
* 정적 타입 지정의 장점은
	* 실행 시점에 어떤 메서드를 호출할지 알아내는 과정이 필요없어 성능이 더 좋음
	* 프로그램 정확성을 컴파일러가 검증하기에 신뢰성이 높음
	* 코드 속 객체가 어떤 타입인지 지정하기에 유지 보수성이 높아짐
	* 더 정확한 코드 완성 기능 등 도구 지원이 더 잘 됨
* Java와 달리 kOltin은 모든 변수의 타입을 개발자가 직접 명시할 필요 없음
* 코틀린 컴파일러가 문맥으로부터 변수 타입을 자동으로 유추해줌(타입 추론)
* 1-2. null 지원 언어
* Kotlin은 Nullable type을 지원함
* 컴파일 시점에 NPE가 발생할 수 있는 지점을 검사할 수 있어서 신뢰성이 높아짐
* 1-3. 함수 타입 지원
* Kotlin은 함수 타입(function type)을 지원함
* 함수형 프로그래밍(functional programming)을 지원하는 언어임

* 2. 함수형 프로그래밍과 객체 지향 프로그래밍
* 함수형 프로그래밍의 핵심 개념은
	* 일급 시민인 함수(first-class) : 함수를 일반 값처럼 다룰 수 있음, 함수를 변수에 저장하거나 함수를 인자로 다른 함수에 전달할 수 있음, 함수에서 새로운 함수를 반환하는 것도 가능함
	* 불변성(immutability) : 일단 만들어지면 내부 상태가 절대로 바뀌지 않는 불변 객체를 사용해 프로그램을 작성함
	* 부수 효과(side effect)없음 : 입력이 같으면 항상 같은 출력을 내놓고 다른 객체의 상태를 변경하지 않으며, 함수 외부나 다른 바깥 환경과 상호작용하지 않는 순수 함수(pure function)을 사용
* 함수형 프로그램의 장점은
	* 간결성
	* 다중 스레드 안정성(safe multithreading)


* 3. Kotlin 코드 컴파일
* Kotlin은 .kt 확장자를 파일에 붙임
* 코틀린 컴파일러는 코틀린 소스코드를 분석해서 .class 파일을 만들어냄
* 만들어진 .class 파일은 개발중인 애플리케이션의 유형에 맞는 표준 패키징 과정을 거쳐 실행됨
* 3-1. Kotlin 빌드 과정
* 코틀린의 빌드 과정은 아래와 같음(mermaid)
	* 코틀린 런타임 -> 애플리케이션
	* .kt -> 코틀린 컴파일러 -> .class -> .jar -> 애플리케이션
	* .java -> 자바 컴파일러 -> .class -> .jar -> 애플리케이션
* 코틀린 컴파일러로 컴파일한 코드(.class)는 코틀린 런타임 라이브러리(kotlin runtime libarary)에 의존함.
* 런타임 라이브러리에는 코틀린 자체 표준 라이브러리 클래스와 코틀린에서 자바 API 의 기능을 확장한 내용이 들어있음
* 코틀린으로 컴파일한 애플리케이션을 배포할때에는 런타임 라이브러리로 함께 배포해야 함
