* 1. Hello, World!
```kotlin
fun main(args: Array<String>) {
	println("Hello, world!")
}
```
* 함수를 선언할 때 fun 키워드 사용
* 파라미터 이름 뒤에 그 파라미터의 타입을 씀
* 변수를 선언할 때에도 마찬가지
* 함수를 최상위 수준에서 정의 가능
	* 자바의 경우 반드시 클래스 안에 함수가 정의되어야 했음
* 배열도 일반적인 클래스와 마찬가지 취급
	* 자바와 달리 배열 처리를 위한 문법이 따로 존재하지 않음
	* (배열 처리를 위한 문법?)
* 코틀린 표준 라이브러리는 자바 표준 라이브러리를 간결하게 사용할 수 있도록 wrapper를 제공함
	* System.out.println -> println
* 세미콜론을 붙이지 않아도 됨

* 2. 함수
```kotlin
fun max(a: Int, b: Int): Int {
	return if(a > b) a else b
}
```
* 함수 선언은 fun 키워드로 시작
* fun 다음에는 함수 이름
* 함수 이름 뒤에는 파라미터 목록
* 함수의 반환 타입은 파라미터 목록의 닫는 괄호 다음에 위치
	* 괄호와 반환 타입 사이를 콜론(:)으로 구분
* kotlin의 if 문의 경우 값을 만들어내지 못하는 문(statement)가 아닌 값을 만들어내는 식(expression)
* 따라서 위 예시처럼 return 문에 결과를 반환하는 형태로 사용 가능

+) 문(statement)과 식(expression)의 구분
식(expression)은 값을 만들어 내며 다른 식의 하위 요소로 계산에 참여 가능
문(statement)은 자신을 둘러싸고 있는 가장 안쪽 블록의 최상위 요소로 존재하며 아무런 값을 만들어내지 않음
Java에서는 모든 제어 구문(ex. if)이 문(statement), 반면 Kotlin에서는 제어 구조가 식(expression)
반면 대입문은 Java에서는 식이지만 Kotlin에서는 문(예시 필요)

* 2-1. 식이 본문인 함수
* 함수의 본문이 식이 되는 것도 가능
```kotlin
fun max(a: Int, b: Int): Int = if(a > b) a else b

// 반환 타입 생략도 가능
fun max(a: Int, b: Int) = if(a > b) a else b
```
* 함수의 중괄호를 없애고 return을 제거하면서 등호(=) 를 붙여 함수의 본문을 식으로 간결하게 표현 가능
* 본문이 중괄호로 둘러싸인 함수를 블록이 본문인 함수
* 등호와 식으로 이뤄진 함수를 식이 본문인 함수라고 함
* 반환 타입 생략도 가능함
	* Kotlin은 Java와 마찬가지로 정적 타입 지정 언어라 반환 타입 생략이 정해져야함
	* 하지만 식이 본문인 함수의 경우 컴파일러가 함수 본문 식을 분석해 식의 결과 타입을 함수 반환 타입으로 정해줌
	* 컴파일러가 타입을 분석해 프로그램 구성 요소의 타입을 정해주는 기능을 타입 추론(Type infrence)라고 함
	* 식이 본문인 함수의 반환 타입만 생략 가능함

* 3. 변수
* Java에서는 변수 선언 시 타입이 맨 앞에 위치함
* 반만 코틀린에서는 앞이 아닌 변수 이름 뒤에 타입을 명시하거나 생략할 수 있음\
```kotlin
val question = "삶, 우주 그리고 모든 것에 대한 궁극적인 질문"
val answer: Int = 42
```
* 타입을 지정하지 않으면 타입 추론을 통해 컴파일러가 변수 타입을 지정해줌
	* 부동소수점(floating point) 상수를 사용한다면 변수 타입은 Double이 됨
* 초기화 식을 사용하지 않고 변수를 선언하려면 변수 타입은 반드시 명시해야함
	* 초기화 식이 없으면 타입 추론이 불가능하기 때문
```kotlin
val answer : Int
answer = 42
```
* 3-1. 변경 가능한 변수와 변경 불가능한 변수
* 변수 선언시 사용하는 키워드는 아래 2개임
		* val : 값을 뜻하는 value, 변경 불가능한(immutable) 참조를 저장하는 변수, Java로 치면 final에 해당
		* var : 변수를 뜻하는 variable, 변경 가능한(mutable) 참조, 자바의 일반 변수에 해당
* 기본적으로 val 키워드를 사용하는 것을 추천
* val 변수는 블록을 실행할 때 정확히 한번만 초기화되어야 함
* 어떤 블록이 실행될 때 오직 한 초기화 문장만 실행됨이 보장된다면, val값을 여러 값으로 초기화 가능
```kotlin
val messsage: String
if(canPerformOperation()) {
	message = "Success"
} else {
	message = "Failed"
}
```
* val 참조 자체는 불변이지만 참조가 가르키는 내부 객체값은 변경 가능
```kotlin
val language = arrayListOf("Java")
language.add("Kotlin")
```
* var 키워드를 사용하면 변수의 값은 변경할 수 있지만, 변수의 타입은 변경되면 안됨
```kotlin
var answer = 42
answer = "no answer" // "Error: type mismatch", 컴파일 에러 발생
```
* 어떤 타입의 변수에 다른 타입의 값을 저장하고 싶다면, 변환 함수를 써서 타입을 변환하거나 값을 변수에 대입할 수 있는 타입으로 강제 형 변환(coerce)해야 함

* 4. 문자열 템플릿
* Kotlin에서 변수를 문자열 안에 사용하고 싶다면 $ 를 사용함
* Java의 문자열 접합 연산(+) 과 동일한 기능이지만 좀 더 간결함
* 컴파일러가 각 식을 정적으로 검사하기 때문에 존재하지 않는 변수를 문자열 템플릿 안에 사용하면 컴파일 오류 발생
* ${} 을 통해 식도 문자열 템플릿 안에 넣을 수 있음
